3 Abordagens Arquiteturais

Para o handoff humano via WhatsApp Business bot por crew

A

WhatsApp-native Recomendado

Tudo via WhatsApp. Sem webhooks externos. O bot recebe uma mensagem WA com contexto e age por conta própria.

Fluxo:
  1. Agente AI chama tool suggest_human_handoff
  2. Quick reply aparece no chat do cliente
  3. Cliente aceita → sistema pergunta número WA (se não conhecido)
  4. Dispatcher envia WA ao cliente: "Transferido! Nossa equipe entrará em contato em breve."
  5. Dispatcher envia WA ao bot da crew: "HANDOFF: Cliente [nome/tel] | Resumo: [IA gera] | Últimas msgs: [transcript]"
  6. Bot reage e entra em contato com o cliente

Pros

  • Zero infra extra
  • Config mínima: só o número
  • Usa dispatcher já existente

Cons

  • Bot parseia texto (menos estruturado)
  • Limite de tamanho da msg WA
B

WA ao cliente + Webhook ao bot

Mensagem WA proativa ao cliente (Flow A) + POST estruturado JSON para URL do bot (Flow B).

Config por crew:
  • humanHandoffWhatsappNumber → número do cliente
  • humanHandoffWebhookUrl → URL que recebe JSON
Payload webhook: { contactName, contactPhone, crewName, summary, transcript[] }

Pros

  • JSON estruturado ao bot
  • Mais robusto p/ n8n/ManyChat
  • Payload ilimitado

Cons

  • 2 configs por crew
  • Bot precisa de servidor HTTP
C

Híbrido com fallback automático

Tenta A (WA nativo) como padrão. Se o crew tiver webhook configurado, usa B adicionalmente.

Lógica:
  1. Sempre envia WA ao cliente (se número disponível)
  2. Sempre envia WA ao bot da crew
  3. Se crew.humanHandoffWebhookUrl existir → também dispara webhook JSON

Permite upgrade progressivo: começa simples (A), adiciona webhook (B) sem mudar o fluxo.

Pros

  • Evolução sem breaking change
  • Config opcional
  • Cobre todos os cenários

Cons

  • Mais código para testar
  • Complexidade incremental
Componentes comuns às 3 abordagens:
  • Campo humanHandoffWhatsappNumber na entidade Crew
  • Tool LLM: suggest_human_handoff(reason)
  • API: POST /api/v1/conversations/:id/human-handoff/accept
  • Novo status de conversa: TRANSFERRED_TO_HUMAN
  • AI para de responder após transferência
  • UI quick replies no widget + fallback "pedir número" + link click-to-chat
