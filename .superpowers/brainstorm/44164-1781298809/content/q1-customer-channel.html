Pergunta 1: Qual o canal do cliente?

Quando o agente sugere transferir para humano, onde está o cliente?

A

Chat Web (Widget)

O cliente está no widget de chat embutido no site/app. A transferência para WhatsApp seria um convite para continuar em outro canal.

Fluxo: Widget → Agente AI sugere → Cliente aceita → Sistema envia mensagem para o WhatsApp Business bot → Bot do WhatsApp entra em contato com o cliente (ou cliente recebe link)
B

WhatsApp (já no canal)

O cliente já está conversando via WhatsApp. A transferência é de um número/bot AI para outro número/bot "humano" da crew.

Fluxo: WhatsApp → Agente AI sugere → Cliente aceita → Sistema transfere para o WhatsApp Business bot da crew → Bot humano continua no mesmo WhatsApp
C

Ambos os canais

A feature deve funcionar tanto para clientes no widget quanto no WhatsApp. O destino é sempre o WhatsApp Business bot da crew.

Fluxo: Qualquer canal → Agente AI sugere → Cliente aceita → Sistema notifica o WhatsApp Business bot da crew → Bot assume o atendimento (cliente migra para WhatsApp se necessário)
Contexto técnico: O sistema já possui webhook de WhatsApp configurável por tenant, dispatcher WhatsApp (Meta Cloud API), e endpoints request-handoff / accept-handoff. A peça nova seria: qual WhatsApp Business bot notificar, e como.
