Como integrar o Test Lab na Crew?

3 abordagens para o simulador de fluxo — escolha a que faz mais sentido para você

A

Aba "Test Lab" na página de detalhe da Crew

Dentro de /dashboard/crews/:id, uma nova aba ao lado das abas existentes. Layout 3 colunas: chat simulado | flow diagram | detail trace.

Prós

  • Tela cheia — espaço para o flow diagram
  • Contexto da Crew sempre visível
  • URL compartilhável para investigar bugs

Contras

  • Requer navegar até a crew primeiro
B

Painel deslizante (Drawer) na listagem de Crews

Botão "Testar" em cada crew card na listagem. Abre um drawer lateral sem sair da página de listagem. Layout 2 colunas: chat | trace/flow.

Prós

  • Acesso rápido sem navegar
  • Pode testar múltiplas crews sem perder contexto

Contras

  • Espaço limitado — flow diagram fica comprimido
  • Difícil mostrar trace completo de admin
C

Página dedicada /dashboard/crews/:id/test

Rota separada, acessível pelo botão "Test Lab" na crew. Layout completo sem sidebar — tela máxima para o flow diagram + chat + trace.

Prós

  • Mais espaço de tela possível
  • URL própria, fácil de bookmarkar
  • Pode exibir histórico de sessões de teste

Contras

  • Sai da navegação principal — menos integrado
  • Mais rotas para manter

Recomendação

A — Aba na página de detalhe da Crew. A crew já tem detalhe (membros, configurações), a aba Test Lab fica naturalmente nesse contexto. Tela cheia garante espaço pro flow diagram e pro trace de admin. A URL /crews/:id?tab=test é compartilhável para investigar bugs específicos.
